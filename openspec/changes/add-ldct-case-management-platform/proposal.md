# 變更提案：LDCT 肺癌篩檢個案管理平台

## Why（為何要改）

院方需集中管理 LDCT 篩檢個案，將健檢名單、影像報告、問卷與人工追蹤資料整合於單一平台，以減少遺漏、標準化追蹤節奏（簡訊／電訪）、並支援結案分類與管理統計。

## What（改什麼）

建立 **LDCT 個案管理系統**，包含：

1. **多來源資料**：健檢名單匯入、LDCT 報告（**自 EHR 取得**；必要時以 **LLM** 從報告文字／文件**擷取結構化欄位**）、個管師手動輸入、問卷系統資料、個管師自 HIS 查詢後登錄之確診資料；**個管師得於本系統連結 HIS 以確認門診／手術等相關資訊**（與手動登錄、追蹤註記搭配）。
2. **個案總清單**：查詢／篩選（含以檢查日期與報告追蹤欄位計算之「本月應追蹤」預設篩選，見 `case-list`、`design.md`）；列尾四連結、檢視報告、結案觸發；**不**另交付獨立「追蹤清單」「追蹤流程」模組（OpenSpec `follow-up-lists`、`follow-up-workflow` 標為 **REMOVED**，與 [計畫書.md](../../../計畫書.md) 第三節一致）。
3. **追蹤紀錄與簡訊**：`follow-up-logging` 登錄／檢視；**簡訊**經遠傳公務機批次發送並記錄（`sms-fetnet`）；不依本變更強制「簡訊／電訪次數與順序」之流程引擎。
4. **結案**：代碼 0–6 分類。
5. **提醒**：病理關鍵字、切片結果、逾期未追蹤等提示。
6. **統計與報表**：本平台提供**報表資料集**，由 **Microsoft Power BI** 製作月報（確診、良性腫瘤、追蹤完成率等）、管理儀表與視覺化（見 `reporting`）。
7. **角色**：個管師、護理師、醫師、管理者之功能權限區隔。
8. **整合邊界**：與 **HIS（個管師連結查詢／確認）**、**EHR（LDCT 報告主來源）**、**LLM（報告欄位擷取）**、**Power BI（報表與視覺化，連線本平台資料集）**、問卷、簡訊之資料流與責任分界（RIS／PACS 由院方經 EHR 彙整者，不強制本平台直接串接）。

## Impact（影響範圍）

- **使用者**：個管師、護理師、醫師、管理者之工作流程與訓練。
- **系統**：需新資料儲存（**Microsoft SQL Server**）、匯入排程或手動匯入、與外部簡訊 API／介面銜接；**雛型**為**單一專案／repo 內前後端分離**（**ASP.NET Core Web API** ＋ **Vue 3**／**TypeScript**）；**生產部署目標**為 **Kubernetes**；**EHR（LDCT 報告）**、**LLM 服務**、**Power BI（報表資料集消費）**、HIS、問卷等可能為批次檔或 API，細節見 `design.md`、[openspec/project.md](../../project.md)。
- **法遵**：個資與醫療資料之存取與稽核需符合院方政策；**LDCT 報告內容若送至 LLM，須符合院方對 PHI 外傳、日誌與模型訓練禁止等規範**。
