# LDCT 肺癌篩檢個案管理系統 — 計畫書

本文件與 OpenSpec 變更 `add-ldct-case-management-platform` 對齊；實作程式或調整範圍時應同步修訂本計畫書與 [openspec/changes/add-ldct-case-management-platform/](openspec/changes/add-ldct-case-management-platform/) 內規格。

## 一、計畫目標

建立 LDCT 篩檢個案追蹤管理平台，整合健檢資料、影像報告、問卷與追蹤紀錄，協助個案管理師管理個案、自動產生追蹤名單、發送簡訊提醒、記錄電訪與追蹤結果、**於系統內連結 HIS 確認相關就醫資訊**、監測確診案例並產生統計報表。

## 二、資料來源


| 來源           | 內容摘要                         |
| ------------ | ---------------------------- |
| 健檢系統名單       | 病歷號、姓名、性別、年齡、檢查日期、客戶名稱、電話、手機 |
| LDCT 報告（EHR，主來源） | 是否有結節、結節數量、最大長度、門診追蹤、3／6／12 個月追蹤；非結構化內容時以 **LLM 擷取** 結構化欄位並供人工覆核 |
| 個管師輸入        | 判讀補充、追蹤狀態、電訪內容、聯繫紀錄          |
| 問卷系統         | 肺癌風險評估問卷                     |
| 個管師（佐證 HIS）  | 經本系統**連結 HIS** 查詢後登錄或註記；確診與病理、手術、Stage 等 |


## 三、核心功能（對應規格 capability）

- 個案清單：`case-list`
- 追蹤清單：`follow-up-lists`
- 追蹤流程：`follow-up-workflow`
- 追蹤紀錄：`follow-up-logging`
- 簡訊：`sms-fetnet`
- 結案：`case-closure`
- 臨床提醒：`clinical-alerts`
- 統計報表：`reporting`
- 角色：`rbac`
- 整合：`system-integrations`、`data-sources`

## 五、風險與假設

- 外部系統（**EHR（LDCT 報告）**、**LLM（欄位擷取）**、HIS、問卷、簡訊）介面與時程依院方與廠商協調；詳見 [design.md](openspec/changes/add-ldct-case-management-platform/design.md)。
- 個資與法遵細節依院方資安核定後補充；**報告內容傳送至 LLM 之 PHI 邊界與禁止訓練用途等須另經核定**。
