# LDCT 個案管理系統 — 系統分析（SA）

## 1. 文件目的與範圍

本文件為 **系統分析（System Analysis）** 層級說明，描述業務背景、利害關係人、需求範圍、資料與外部介面（業務觀點），以及非功能需求摘要。**驗收細節與行為規格**以 OpenSpec 變更為準（見第 9 節）。

## 2. 背景與問題

院方需集中管理 **LDCT（低劑量肺部電腦斷層）篩檢個案**，將健檢名單、報告、問卷與人工追蹤資料納入單一流程，以降低遺漏、標準化追蹤節奏（簡訊／電訪）、支援結案分類與管理統計。

## 3. 利害關係人與角色

| 角色 | 主要訴求（業務） |
|------|------------------|
| 個案管理師 | 追蹤名單、電訪與聯繫紀錄、結案、確診資料登錄、**連結 HIS 確認就醫資訊**、簡訊（依授權） |
| 護理師 | 肺癌風險評估問卷（平板或介接問卷系統） |
| 醫師 | 報告判讀、補充說明（依院方流程） |
| 管理者 | 統計與報表（透過 **Power BI** 與本平台提供之資料集） |
| 資訊／資安 | 介接、身分驗證、個資與稽核、LLM／雲端邊界 |

## 4. 業務目標（摘要）

- 管理 LDCT 篩檢個案並提供總清單與查詢／篩選。
- 依報告與規則**自動產生**追蹤分類（胸腔門診、3／6／12 個月等）與追蹤清單。
- 支援既定追蹤流程（簡訊、電訪次數與順序）與**追蹤紀錄**。
- 整合**簡訊平台**（遠傳公務機）批次發送並留紀錄。
- **結案代碼** 0–6 分類與統計口徑一致。
- **臨床提醒**（病理關鍵字、切片、逾期等）。
- **統計與報表**：由本平台提供**報表資料集**，以 **Microsoft Power BI** 製作月報與管理儀表。

## 6. 資料需求（來源與責任）

| 來源 | 業務內容（摘要） |
|------|------------------|
| 健檢／健管名單 | 病歷號、姓名、性別、年齡、檢查日期、客戶名稱、電話、手機 |
| EHR（LDCT 報告） | 結節與追蹤建議等欄位；非結構化時經 **LLM 擷取** 並供人工覆核 |
| 個管師輸入 | 判讀補充、追蹤狀態、電訪與聯繫紀錄 |
| 問卷系統 | 肺癌風險評估問卷 |
| HIS（查詢） | 個管經**連結 HIS** 查詢後，於本平台登錄或註記；確診與病理等 |
| **本平台** | 營運資料、追蹤紀錄、簡訊紀錄、結案狀態；**報表資料集**供 Power BI |

## 7. 外部系統（業務介面）

| 系統 | 業務關係 |
|------|----------|
| EHR | LDCT 報告欄位之主取得來源 |
| HIS | 個管自本平台**連結**後查詢；權威在 HIS |
| LLM | 報告文字／文件之結構化擷取（須資安核定） |
| 問卷系統 | 問卷結果匯入或 API |
| 遠傳公務機簡訊 | 批次簡訊與發送紀錄 |
| Power BI | 連線報表資料集，產製月報與儀表板（非 LDCT 報告匯入來源） |

## 8. 非功能需求（摘要）

| 類別 | 說明 |
|------|------|
| 資安 | 身分驗證、授權、稽核、個資與 PHI；LLM 傳輸與禁止訓練等依院方核定 |
| 法遵 | 個資法與院方醫療資訊政策 |
| 可用性／效能 | 待實作階段訂定量化指標（如回應時間、併發數） |
| 備援 | 主機與備份 RPO／RTO 待決議（見 [SD.md](SD.md)） |

- 外部系統介面與時程依院方與廠商協調；未確定前以 **TBD** 標示。
